Key HTTP Concepts

Understanding the following concepts makes it easier to read browser developer tools, design APIs, troubleshoot websites, and reason about web performance.

URLs

A URL can contain a scheme, host, port, path, query string, and fragment. Together these parts identify how and where a resource should be accessed.

Methods

GET retrieves a representation, POST submits data, PUT replaces or creates a resource, PATCH applies a partial change, and DELETE requests removal.

Status Codes

Codes are grouped by class: 1xx informational, 2xx successful, 3xx redirection, 4xx client error, and 5xx server error.

Headers

Headers carry metadata such as content type, accepted formats, authentication information, cache rules, and cookie instructions.

Caching

Caching can reduce repeated downloads. Cache-Control, validators such as ETag, and conditional requests help determine whether stored content can be reused.

Cookies and Sessions

Servers can ask browsers to store small cookie values. Cookies may support sessions, preferences, and authentication, but require careful security and privacy controls.

Content Negotiation

A client can describe acceptable media types or languages, and the server may select an appropriate representation.

APIs and JSON

Many web APIs use HTTP methods and status codes while exchanging JSON data rather than complete HTML pages.
