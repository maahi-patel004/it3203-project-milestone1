Key Concepts

The concepts below are important for understanding how HTTP works in real websites and applications.

  1. Client-server model: A client sends requests, and a server returns responses.
  2. URL: A web address that identifies where a resource is located.
  3. HTTP methods: Actions such as GET, POST, PUT, and DELETE that describe the purpose of a request.
  4. Status codes: Numbered messages such as 200, 404, and 500 that summarize the result of a request.
  5. Headers: Extra information sent with requests and responses, such as content type or caching instructions.
  6. Caching: A way to store copies of resources so pages can load faster and reduce server work.
  7. HTTPS: The secure version of HTTP that uses encryption to protect data.
