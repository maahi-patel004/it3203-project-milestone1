What Is HTTP?

HTTP is an application-layer protocol used to transfer representations of resources. A resource can be an HTML document, image, video, stylesheet, script, or data returned by an API. HTTP follows a client-server model: the client initiates a request, and the server returns a response.

Key idea: HTTP is stateless. Each request is interpreted independently unless state is added through mechanisms such as cookies, sessions, or authentication tokens.

1. The Client Creates a Request

The browser reads a URL, determines the host and path, and sends a request. The request includes a method, target, protocol version, headers, and sometimes a body.

2. The Server Processes It

The server receives the message, checks routing and permissions, performs required application work, and prepares a response.

3. The Server Responds

The response contains a status code, response headers, and usually content. A 200 code commonly means success, while 404 means a resource was not found.

4. The Client Renders the Result

The browser interprets HTML and retrieves linked CSS, JavaScript, fonts, and images through additional requests.
