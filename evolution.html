Evolution of HTTP

HTTP evolved as the web changed from a collection of simple documents into a platform for interactive applications, streaming, commerce, and real-time services. Each major version tried to reduce delay, use connections more efficiently, and support modern network conditions.

1991

HTTP/0.9

The original version supported a very simple GET request and returned only an HTML document. It did not include headers or status codes.

1996

HTTP/1.0

HTTP/1.0 introduced request and response headers, status codes, content types, and support for resources beyond HTML.

1997+

HTTP/1.1

Persistent connections became the default, reducing the need to create a new TCP connection for every resource. Host headers enabled multiple websites to share one server address, and caching controls became more capable.

2015

HTTP/2

HTTP/2 uses a binary framing layer and multiplexing so multiple request and response streams can share one connection. Header compression also reduces repeated metadata.

2022

HTTP/3

HTTP/3 maps HTTP semantics over QUIC instead of TCP. QUIC integrates transport security and reduces the effect of packet loss on unrelated streams, which can help on mobile or unstable networks.
