HTTP, HTTPS, and Web Security

Why Plain HTTP Is Not Enough

Plain HTTP does not encrypt the messages traveling across the network. Someone able to observe or alter the connection may read sensitive information or modify content. HTTPS protects the HTTP exchange by using Transport Layer Security (TLS).

Confidentiality

Encryption makes intercepted traffic difficult to understand without the session keys.

Integrity

TLS helps detect whether data was modified while moving between the client and server.

Authentication

Digital certificates help a browser verify that it is communicating with the intended domain.

How an HTTPS Connection Is Established

  1. The client connects to the server and proposes supported TLS options.
  2. The server sends its selected options and digital certificate.
  3. The client validates the certificate's domain, signature, and validity period.
  4. The parties establish shared session keys.
  5. HTTP messages are exchanged through the encrypted TLS connection.
